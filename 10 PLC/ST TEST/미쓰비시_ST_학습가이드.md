# 미쓰비시 PLC 실무자를 위한 ST 언어(Structured Text) 학습 가이드

현업에서 래더(Ladder)를 자유롭게 다루시는 프로그래머라면, 백지상태의 초보자보다 ST를 훨씬 빠르게 습득하실 수 있습니다. **어차피 메모리 구조(D, M, X, Y 등)와 PLC의 동작 원리(스캔 타임 등)를 완벽히 꿰뚫고 계시기 때문**입니다. 문법만 배우시면 됩니다!

---

## 1. 마인드셋: "모든 것을 ST로 짤 필요는 없다"

실무에서 가장 추천하는 방식은 래더와 ST를 섞어 쓰는 **혼합형(Hybrid) 설계**입니다. 유지보수성을 위해 이것만 명심하세요.

- **래더(Ladder)가 유리한 곳:** 장비의 시퀀스 제어, 인터록 조건, 즉각적인 I/O 모니터링 (눈으로 살아서 움직이는 걸 봐야 하는 곳)
- **ST가 압도적으로 편한 곳:** 복잡한 사칙/삼각함수 연산, 배열(Array) 데이터 일괄 처리, 문자열(String) 조작, 바코드 통신 파싱 등

---

## 2. 실전 학습 4단계 로드맵

### 🟢 1단계: GX Works3의 '인라인 ST (Inline ST)' 기능부터 쓰기

처음부터 ST 전용 POU(프로그램)를 만들면 헷갈립니다. 래더 창 중간에 네모난 박스로 ST 블록을 삽입할 수 있는 **인라인 ST** 기능을 적극 활용하세요.

- **연습 목표:** 래더에서 번거롭게 산술 연산(`ADD`, `SUB`, `MUL`, `DIV`)하던 부분을, 인라인 ST 박스 안에서 수학 공식처럼 한 줄로 바꿔봅니다.
- **코드 예시:**
  ```pascal
  // 여러 개의 래더 블록이 이 한 줄로 끝납니다.
  D100 := (D200 * 10) / D300;
  ```

### 🟡 2단계: 머릿속 래더를 ST 코드로 번역해 보기

타이머나 카운터를 쓰지 않는 기본적인 로직을 `IF` 나 `=` 연산자로 변환해 보며 언어의 감을 잡습니다.

**1. A접점 출력 전환**

```javascript
// 래더: X0이 살면 Y0이 산다.
IF X0 THEN
    Y0 := TRUE;
ELSE
    Y0 := FALSE;
END_IF;

// 현업 고수들의 축약형 표현 (대입 방식)
Y0 := X0;
```

**2. 자기유지 회로 만들기**

```pascal
// 래더: 기동(X0) 누르면 유지, 정지(X1) 누르면 해제
Y10 := (X0 OR Y10) AND NOT X1;
```

### 🟠 3단계: 제어용 펑션블록(FB)을 통째로 ST로 설계하기

아날로그 센서 스케일 변환이나 특정 모터 제어 등을 수행하는 **공용 펑션블록(FB)**을 만드실 때, 내부에 래더 대신 ST를 적어보세요.

- 이렇게 만들어둔 ST 펑션블록은 나중에 다른 미쓰비시 프로젝트에 라이브러리로 쉽게 이관하여 우려먹기(!) 아주 좋습니다.

### 🔴 4단계: FOR 반복문과 인덱스 Z 레지스터 대체하기

ST 언어를 배우는 가장 큰 이유입니다. 래더에서 포인터(`Z0`, `Z1`)를 쓰며 헷갈리게 데이터 블록을 밀거나 지우던 것을 ST의 배열과 `FOR` 문으로 우아하게 해결하세요.

```pascal
// 연습: D1000 ~ D1099 영역(100개 워드)을 모두 0으로 초기화
FOR i := 0 TO 99 DO
    D[1000 + i] := 0;
END_FOR;
```
